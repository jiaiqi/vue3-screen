🔍 拖拽系统深度选型分析

设计器中存在 5 类完全不同的拖拽场景，没有任何单一库能完美覆盖所有场景，需要分场景选型。

❌ vue-draggable-plus 不适合作主力

vue-draggable-plus 基于 Sortable.js，本质是一个列表排序库。核心限制：

  • 不支持画布内绝对定位的自由移动（只能排序）
  • 多选拖拽支持弱，无法批量移动任意位置的元素
  • 坐标计算逻辑不透明，很难精确控制落点坐标转换
  • 与 Vue Flow 的 drag 事件系统冲突
  • Ghost 节点样式定制能力有限
仅列表排序不适合画布

✅ 推荐方案：分场景多策略

针对 5 类场景分别选用最合适的方案，在架构层统一封装为 useDrag composable：

  • 物料→画布拖入：原生 HTML5 DragDrop API（最轻量，跨容器无需安装库）
  • 画布内自由移动：VueUse useDraggable + Pointer Events（精确坐标，多选支持）
  • 图层/属性排序列表：vue-draggable-plus（列表排序正是其强项）
  • 图元连线拖出：Vue Flow 内置 Handle 拖拽系统（与图引擎深度集成）
  • 画布缩放调整手柄：Interactjs resizable（专业的 resize 手势处理）

统一封装层设计
所有拖拽操作封装在 packages/core/composables/useDrag.ts 中对外暴露统一接口。 外部调用无需关心底层使用哪个库，方便未来替换。所有拖拽操作结束时统一触发 Command 提交到 HistoryManager，确保 Undo/Redo 正常工作。 拖拽过程中使用 requestAnimationFrame 节流，防止高频更新导致卡顿。

三大架构原则
关注点分离：设计态与运行态完全隔离，Schema 为唯一交换格式。混合画布的 Widget 节点和 Graph 节点使用同一 Schema 根，通过 nodeKind 字段区分，渲染器按类型分发。
双层渲染：DOM 层负责所有节点渲染（Widget + Graph），SVG 层专注连线和动画，两层共享同一逻辑坐标系，无缝叠加。不替换现有 CSS 绝对定位体系，零破坏性。
插件优先：核心最小化，组件库、图元库、数据源类型、动画类型、工具栏按钮均通过插件注入。企业定制化需求 100% 通过插件满足，不修改核心代码。

DESIGNER CORE ENGINE

4.1 画布引擎 (Canvas Engine)

基于 CSS Transform 的坐标系管理，针对大屏高分辨率做了专项优化。混合画布模式下，DOM 节点层与 SVG 连线层共享同一变换矩阵。

canvas-engine.ts
interface CanvasConfig { width: number // 设计宽度 e.g. 1920 height: number // 设计高度 e.g. 1080 scale: number // 当前缩放比例 offsetX: number // 水平平移偏移 offsetY: number // 垂直平移偏移 bgColor: string // 背景色 bgImage?: string // 背景图 URL bgImageOpacity: number // 背景图透明度蒙版 grid: GridConfig // 网格吸附 rulers: boolean // 标尺显示 canvasMode: 'screen'|'hybrid'|'topology' } // 画布变换矩阵（两层共享） const transform = computed(() => `translate(${offsetX}px,${offsetY}px) scale(${scale})` )
  • 自适应缩放：fit-screen / 1:1 / 自定义，保持相对坐标不变
  • 标尺与辅助线：水平/垂直标尺，拖出辅助线，吸附对齐，支持辅助线管理面板
  • 多分辨率预设：1920×1080、2560×1440、3840×2160 等
  • 背景图蒙版：透明度可调，防止背景图干扰编辑操作
  • 鸟瞰图导航：右下角缩略图，点击快速定位大画布区域

4.2 拖拽系统 (DnD System)

分场景多策略，统一封装。物料拖入采用原生 DnD，画布内移动使用 VueUse useDraggable，列表排序使用 vue-draggable-plus。

物料拖入画布 — 完整流程

dragstart → dataTransfer 携带 {type, meta}
↓
dragover → 实时显示 Ghost 预览，计算落点
↓
drop → 坐标转换（屏幕坐标→画布逻辑坐标）
↓
吸附计算（网格/辅助线/边缘对齐）
↓
AddNodeCommand 写入 Schema → 历史栈
  • 多选拖拽：Ctrl+点击 / 框选，批量移动，相对位置保持
  • 智能吸附：边缘对齐、中心对齐、间距均等辅助线
  • 容器感知：拖入容器组件时自动切换坐标系（绝对→相对）
  • 键盘微调：方向键 1px，Shift+方向键 10px 精确位移
  • 触摸支持：Pointer Events API 同时处理鼠标和触控

4.3 属性面板系统 (Props Panel)

基于 JSON Schema 自动生成表单，支持 40+ 种控件类型。混合画布模式下，选中连线时属性面板切换为连线配置面板。

属性配置

数值输入、颜色选择器（支持渐变）、字体选择、下拉、开关、代码编辑器（Monaco）、图片上传

样式配置

尺寸/位置、背景（颜色/图片/渐变）、边框、阴影、透明度、混合模式、CSS 滤镜、动画

数据绑定

选数据源 → JSONPath 字段提取 → 过滤器 → 字段映射 → 刷新频率，连线额外有动画数据绑定

4.4 历史记录系统 (History)

基于 Command Pattern 实现，每个操作封装为可逆命令。混合画布新增图操作命令类型。

history-commands.ts
abstract class BaseCommand { abstract execute(): void abstract undo(): void description: string } // Widget 操作 class AddWidgetCommand extends BaseCommand {} class MoveCommand extends BaseCommand {} class ResizeCommand extends BaseCommand {} // 拓扑操作（新增） class AddGraphNodeCommand extends BaseCommand {} class AddEdgeCommand extends BaseCommand {} class UpdateEdgeAnimationCommand extends BaseCommand {} // 批量操作 class BatchCommand extends BaseCommand { commands: BaseCommand[] // 多操作一步撤销 }

4.5 图层面板 (Layer Panel)

树形结构展示所有节点，支持嵌套容器的折叠展开。Widget 节点和 Graph 节点在同一图层树中显示，通过图标区分类型。

  • 双击图层名称内联编辑节点名称
  • 图层锁定：锁定后画布无法选中该节点
  • 图层隐藏：设计态隐藏（运行时仍显示）
  • 批量操作：多选后统一设置锁定/隐藏/透明度
  • vue-draggable-plus 实现图层排序（改变 zIndex）
  • 图层组：Ctrl+G 将多个节点打组，组内坐标相对
  • 容器节点：树形折叠展开，子节点缩进显示
  • 搜索过滤：输入关键字高亮匹配图层
  • 图层颜色标注：为图层设置颜色便于区分
  • 连线图层：连线在图层树中也有对应条目，可重命名

混合画布架构决策
HYBRID CANVAS ARCHITECTURE

将数据大屏 Widget 与工业拓扑图元融合在同一画布，面临两套截然不同渲染范式的深度整合。

❌ 方案A：单一图引擎

将所有组件迁入 Vue Flow / AntV G6，图引擎统一管理

破坏性重构ECharts 集成复杂性能风险

✅ 方案B：双层混合渲染（采用）

DOM 层负责所有节点（Widget+Graph），SVG 层专注连线动画，共享坐标系

零破坏性性能最优渐进增强

⚠️ 方案C：iframe 隔离

拓扑子画布用 iframe 嵌入，postMessage 通信

坐标不统一交互割裂

坐标系统一原则
四层渲染共享同一逻辑坐标系（由画布引擎维护的 scale + translate 矩阵）。SVG 连线层的 viewBox 始终与画布实际像素尺寸同步。 所有节点坐标、连接点位置均在同一坐标空间计算，无论画布如何缩放平移，连线端点始终精确对准 Port 位置。 混合画布模式下，Vue Flow 的节点坐标系与画布引擎坐标系通过转换函数保持同步。

Edge Schema 完整定义

edge-schema.ts
interface EdgeSchema { id: string source: string // 源节点 ID sourcePort: string // 源 Port ID target: string targetPort: string pathType: 'bezier'|'straight'|'step'| 'smoothstep'|'orthogonal' style: { stroke: string strokeWidth: number opacity: number } animation: EdgeAnimation label?: { text: string dataBinding?: DataBinding position: number // 0~1 沿路径位置 align: 'center'|'source'|'target' } }

路径类型预览

bezier 贝塞尔曲线（默认）
straight 直线
step 阶梯折线
orthogonal ★ 正交避障（工业 P&ID 首选）
smoothstep 平滑阶梯
正交路径 A* 避障
工业 P&ID 图要求所有连线只走水平/垂直方向。采用 A* 算法在逻辑网格上计算最短正交路径，自动绕过其他节点， 路径弯折点用 SVG 二次贝塞尔曲线处理圆角，最终生成平滑的正交连线。A* 计算在 Web Worker 中执行不阻塞主线程。

interface EdgeAnimation { type: 'none'|'waterFlow'|'electric'|'arrowFlow'|'particle'|'pulse'|'custom' speed: number // 0.1~5.0 direction: 'forward'|'reverse'|'auto' color: string opacity: number // 管道专用 pipeWidth?: number // 管道外径 fluidWidth?: number // 流体宽度 pipeColor?: string // 管道外壳颜色 showPipe?: boolean // 粒子专用 particleCount?: number particleSize?: number particleGlow?: boolean // ★ 数据驱动动画（工业核心） dataBinding?: { speedField?: string // 哪个字段控制速度 dirField?: string // 哪个字段控制方向（正负） colorField?: string // 哪个字段控制颜色（温度映射） activeField?: string // 哪个字段控制开关（阀门状态） colorMap?: {value:number, color:string}[] // 值→颜色映射 } }
数据驱动动画典型映射
流速控制：流量值 0~100 → 动画速度 0.2~5.0 线性映射，流量为 0 时动画自动暂停
方向控制：流量方向字段 > 0 → forward；< 0 → reverse，实现双向流体
温度映射：温度字段通过 colorMap 插值，低温→蓝色，高温→红色，渐变过渡
阀门联动：阀门关闭状态为 true → animation.type = 'none'，管道变灰
故障告警：设备故障 → 连线动画强制变红色闪烁，忽略原色彩配置
电压状态：电压正常 → 绿色正常频率；欠压 → 黄色；断电 → 无动画灰色

性能优化策略
PERFORMANCE OPTIMIZATION

设计态优化

  • 虚拟化渲染：节点超 200 个时启动，仅渲染视口内节点
  • 拖拽 Ghost：拖拽中仅移动 Ghost，drop 时才更新 Schema
  • 属性防抖：输入框 300ms 防抖，避免频繁 Schema 更新
  • Worker 计算：吸附算法、A* 路径、碰撞检测在 Web Worker
  • 组件懒加载：物料面板按分类懒加载

运行态优化

  • 代码分割：按组件类型拆包，发布端按需加载
  • 数据节流：WebSocket 高频数据按帧节流，最高 60fps
  • ECharts 增量：setOption + notMerge:false 避免重建
  • 内存管理：组件销毁时清理定时器、取消数据订阅
  • IndexedDB 缓存：Schema 和静态数据本地缓存

混合画布优化

  • 纯 CSS 动画：不占 JS 主线程，GPU 加速
  • 视口外暂停：IntersectionObserver 暂停非可见动画
  • 边路径缓存：节点未移动时路径字符串 memoize
  • Canvas 降级：边超 500 条时静态边用 Canvas 渲染
  • Worker 布局：dagre 自动布局在 Web Worker 计算

嵌入 SDK 设计
EMBED SDK

提供 @screen/embed-sdk 包，支持将大屏渲染器嵌入任意 Web 应用，无需 iframe，直接以 Web Component 或 Vue 组件形式集成。

embed-sdk 使用示例
// 方式1：Web Component（框架无关） import '@screen/embed-sdk' // HTML: <screen-renderer screen-id="xxx" token="yyy" theme="dark"></screen-renderer> // 方式2：Vue 组件 import { ScreenRenderer } from '@screen/embed-sdk/vue' // <ScreenRenderer :screenId="id" :token="token" :variables="vars" @event="onEvent"/> // 方式3：JS API const renderer = new ScreenRenderer({ container: '#app', screenId: 'screen_xxx', token: 'jwt_token', variables: { siteId: '001' }, // 注入全局变量 onEvent: (eventName, data) => {} // 监听大屏事件 }) renderer.setVariables({ timeRange: '7d' }) // 运行时修改变量 renderer.destroy() // 销毁释放资源
SDK 安全机制
嵌入 Token 由服务端签发（JWT），包含屏幕 ID、有效期、允许嵌入的 Referer 域名列表。 SDK 初始化时向服务端验证 Token 合法性，每次数据请求都携带 Token，防止未授权访问。 支持 Token 自动续期，确保长时间展示的大屏不会因 Token 过期而中断。

主题引擎与全局样式
THEME ENGINE

工业级大屏需要极强的样式定制化能力，方案采用 CSS 变量定义契约 + UnoCSS 原子化编排 结合的双轨驱动模型。

CSS Theme Tokens

抽取超过 200 个全局设计 Token（色彩、响应式断点、阴影深浅），全部映射为 CSS Variables，支持黑暗/明亮以及专属品牌套件（例如科技蓝主题、暗夜工业主题）。

UnoCSS 深度集成

基于 UnoCSS 实现构建时的组件样式按需提取；通过 @unocss/preset-icons 引入纯 CSS 图标加载机制，将 iconify 上的海量工业图标映射为 HTML 类名。

组件深层覆盖

Element Plus 的组件库及第三方图表（ECharts），均在顶层容器捕获 Token 值。动态修改 document.documentElement.style 即可在一毫秒内完成数百个组件的换肤刷新。

多人协同架构设计 (预研)
COLLABORATION ARCHITECTURE

针对企业级用户多人同时编辑同一个工业组态大屏的诉求，基于 Y.js (CRDT 算法) 实现云端状态分发与操作冲突自动解决。

协同模块技术方案解决的核心痛点
状态同步Y.js + y-websocket基于 CRDT 算法提供无锁并发的数据结构同步，比 OT 算法更容易实现，天然支持去中心化或服务端转发。
冲突解决增量指令队列多用户同时移动 Vue Flow 图元或编辑同一文本，依靠数组的相对索引修改合并差异，不会发生互相覆盖的问题。
用户感知感知Yjs Awareness Protocol实时获取其他协作者的光标位置、选中的节点并以不同的色彩高亮标注（例如显示 "王工 正在编辑水泵"）。
离线编辑IndexedDB 本地持久化缓存协同断网时退化为本地缓存栈，网络恢复后，自动与服务器通过协同差异向量(State Vector)进行快照合并。

测试与质量保障体系
TESTING & QA

工业控制、物联大屏系统对代码质量有严苛要求。结合 Monorepo 体系落实三位一体的测试金字塔：

工程化 CI 质量拦截流
commit 触发 → Lint 静态验证 → Vitest 核心用例评估 → E2E 视图快照回归比对
  • 引擎级单元测试 (Vitest)：针对 Command 历史记录队列、 Schema 解析映射、 A* 寻路避障算法等核心 JavaScript 计算逻辑进行输入输出比对，指标要求分支覆盖率 > 80%。
  • E2E 集成测试 (Playwright)：涵盖画布操作的跨浏览器真实场景模拟（拖拽物料入画布、图元连接 Port 点、右键调出属性菜单并修改图表颜色），用于重构或迭代时的主流程护航。
  • 视觉回归测试 (Visual Regression)：借助 Playwright Snapshot，以像素对比级记录边动画渲染（Edge Animations）和复杂渐变图表的视觉差异并自动上报。
